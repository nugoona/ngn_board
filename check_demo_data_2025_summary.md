월간 리포트 생성 시 `bq_monthly_snapshot.py`의 `run()` 함수는 다음과 같은 순서로 데이터를 조회합니다:

1. **GCS 스냅샷 확인** (있으면 사용)
2. **BigQuery에서 데이터 조회**:
   - `daily_cafe24_sales` - ❌ 2025년 데이터 없음
   - `mall_sales_monthly` - ✅ 5월~12월 있음
   - `meta_ads_account_summary` - ✅ 5월~12월 있음
   - `ga4_traffic_ngn` - ✅ 1월~12월 있음

**결론**: `daily_cafe24_sales` 테이블에 2025년 데이터가 없어서 월간 리포트가 생성되지 않습니다.

### 해결 방법

1. **데이터 확인**: `daily_cafe24_sales` 테이블에 demo 계정의 2025년 데이터가 실제로 없는지 확인
2. **데이터 수집**: demo 계정의 2025년 카페24 매출 데이터를 수집해야 함
3. **월간 집계 활용**: `mall_sales_monthly`에 데이터가 있으므로, 해당 테이블을 우선 활용하는 방법도 고려 가능
